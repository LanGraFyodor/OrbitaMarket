# Выполнение технического задания итогового проекта

Источник обязательных требований — техническое задание LMS `Индустриальная разработка ПО: Итоговый проект по программе`. Сначала показано выполнение требований курса, затем отдельно перечислены дополнительные инженерные возможности, которые я реализовал для развития проекта до более цельного продукта.

Статус **ВЫПОЛНЕНО** означает, что реализация находится в репозитории и проверяется сборкой, unit- или системным тестом. PDF, презентация и публикация отдельного репозитория автотестов оформляются отдельно от программного кода.

## 1. Выполнение обязательных требований

### 1.2. Payments Service

| Требование | Моя реализация | Проверка | Статус |
|---|---|---|---|
| Создать один счёт на `user_id` | Unique constraint и идемпотентный `POST /accounts`; повтор не создаёт второй счёт | unit + E2E | **ВЫПОЛНЕНО** |
| Пополнить счёт на `amount > 0` | Серверная валидация, `INVALID_AMOUNT`, атомарный SQL credit | API/E2E | **ВЫПОЛНЕНО** |
| Получить неотрицательный целочисленный баланс | Value response в геокредитах; отрицательный баланс запрещён SQL-условием | API/unit | **ВЫПОЛНЕНО** |
| Асинхронно списать сумму заказа | Consumer принимает `OrderPaymentRequested`; HTTP создания заказа не ждёт оплату | Kafka E2E | **ВЫПОЛНЕНО** |
| Идемпотентность списания по `order_id` | Unique payment `order_id`, inbox `event_id`, повтор команды не уменьшает баланс второй раз | E2E scenario 3 | **ВЫПОЛНЕНО** |
| Корректность при конкуренции | `UPDATE ... SET balance = balance - ? WHERE balance >= ?`; потерянных обновлений и отрицательного баланса нет | unit/E2E scenario 4 | **ВЫПОЛНЕНО** |

### 1.3. Orders Service

| Требование | Моя реализация | Проверка | Статус |
|---|---|---|---|

- экспорт C4 C1/C2 в PDF;

- публикация каталога `autotests` отдельным публичным Git-репозиторием;
- повторный Gitleaks/Semgrep на финальном commit перед защитой.

Программная реализация, конфигурация и проверочные сценарии находятся в этом репозитории; перечисленные материалы относятся к оформлению защиты и публикации.
