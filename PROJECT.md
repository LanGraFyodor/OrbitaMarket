# PROJECT — OrbitaMarket

## Цель

Создать MVP платформы БЮРО 1440, которая принимает заказы спутниковых продуктов, асинхронно и идемпотентно списывает геокредиты при конкурентной нагрузке и предоставляет пользователю готовый учебный геопродукт по AOI.

## Стейкхолдеры

1. **Оператор ДЗЗ** — создаёт tasking/monitoring-заказы и контролирует их статусы.
2. **Аналитик/заказчик** — покупает архивные снимки, выделяет AOI и получает продукт.
3. **Владелец платформы/администратор** — отвечает за тарифы, доступность, безопасность и отчётность.

## Границы ответственности

- Gateway маршрутизирует HTTP и формирует доверенный `X-User-Id` из JWT, но не хранит бизнес-состояние.
- Orders владеет заказом, outbox команды оплаты и применением результата.
- Payments владеет единственным счётом, балансом и идемпотентным списанием.
- Auth владеет учётными данными и профилем.
- Notifications владеет историей уведомлений и SSE.
- Rust Geo Service владеет вычислением AOI, тарифом и imagery fulfillment.
- Одна физическая PostgreSQL используется только для удобства Compose; схемы и таблицы сервисов изолированы.

## Критерии MVP

- пять обязательных сценариев раздела 7.1 воспроизводимы через Gateway;
- повтор события с тем же `order_id` не уменьшает баланс второй раз;
- конкурентные заказы не делают баланс отрицательным;
- все обязательные REST endpoints имеют успешный системный тест;
- проект поднимается одной командой Compose;
- C1/C2, SQL, чек-лист и security triage находятся в репозитории.
